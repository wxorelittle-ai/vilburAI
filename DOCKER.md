# Бригадир.Про — запуск через Docker

Самый надёжный способ запустить приложение, если на компьютере VPN/прокси
(V2Ray, Clash, Shadowsocks и т.п.) мешает обычной установке через `pip`
(ошибка вида `Missing dependencies for SOCKS support`). Docker Desktop
собирает и запускает приложение внутри изолированной Linux-среды со своей
сетью — в большинстве случаев это полностью обходит проблему с прокси
на хосте.

## Шаг 1. Установите Docker Desktop

Скачайте и установите с [docker.com/products/docker-desktop](https://www.docker.com/products/docker-desktop/)
(Windows/macOS/Linux). После установки запустите Docker Desktop и дождитесь,
пока в трее появится иконка кита без анимации загрузки — значит, движок готов.

## Шаг 2. Запустите приложение

Откройте PowerShell (Windows) или Терминал (Mac/Linux) в папке проекта
(`brigadir_pro`) и выполните:

```
docker compose up --build
```

Первый запуск займёт несколько минут (скачивание образа Python и установка
зависимостей внутри контейнера). В консоли должна появиться строка:

```
Starting Brigadir.Pro on http://0.0.0.0:8000/
...
Starting development server at http://0.0.0.0:8000/
```

## Шаг 3. Откройте приложение

В браузере: **http://localhost:8000**

Вход в админку (`/admin/`) создаётся автоматически при первом запуске:
- Логин: `admin`
- Пароль: `admin123`

(Это локальный запуск на вашем компьютере, наружу не открыт — но если вы
всё же выставляете его в интернет, обязательно смените пароль администратора.)

## Остановка и повторный запуск

Остановить — `Ctrl+C` в том же окне терминала, либо в новом окне:
```
docker compose down
```

Запустить снова (без пересборки, если код не менялся):
```
docker compose up
```

Данные (бригады, документы, сметы) сохраняются между запусками — они лежат
в Docker-томе `sqlite_data`, который не удаляется при остановке контейнера.
Логотипы и другие загруженные файлы сохраняются в папке `media/` рядом
с проектом.

## Если Docker тоже не подключается к интернету при сборке

Значит прокси настроен и на уровне самого Docker Desktop. Проверьте:
**Docker Desktop → Settings → Resources → Proxies** — там либо выключите
прокси, либо укажите тот же прокси, что использует Windows (тогда сборка
пойдёт через него, но уже с корректной поддержкой, в отличие от голого pip).

## Чем это отличается от `deploy/` конфигурации

Эта Docker-настройка — для быстрого локального запуска одним человеком на
своём компьютере (SQLite, `DEBUG=True`, сервер разработки Django). Для
боевого сервера с несколькими пользователями используйте полноценный деплой
из `deploy/DEPLOY.md` (PostgreSQL, Gunicorn, Nginx, SSL).
