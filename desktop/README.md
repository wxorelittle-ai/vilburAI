# Вильбур AI — десктоп-версия

Локальный запуск приложения на вашем компьютере одним файлом, без сервера
и без интернета (после установки). Данные (бригады, документы, расчёты)
хранятся в файле `db.sqlite3` рядом с проектом и никуда не отправляются.

## Windows

1. Установите Python 3.11+ с [python.org](https://www.python.org/downloads/) —
   при установке обязательно поставьте галочку **Add python.exe to PATH**.
2. Распакуйте архив проекта в любую папку.
3. Дважды кликните **`install_windows.bat`**.
   Скрипт сам поставит всё нужное и запустит приложение — это займёт пару минут
   при первом запуске.
4. В следующий раз просто запускайте **`run_desktop.bat`** — он не переустанавливает
   зависимости заново, поэтому стартует за секунды.

## macOS / Linux

1. Откройте Терминал в папке с проектом.
2. Запустите:
   ```bash
   ./install_mac_linux.sh
   ```
3. В следующий раз:
   ```bash
   ./run_desktop.sh
   ```

На macOS нативное окно приложения работает «из коробки» (системный WebKit).
На Linux для нативного окна нужен системный WebKitGTK
(`sudo apt install python3-gi gir1.2-webkit2-4.1` на Ubuntu/Debian) — без него
приложение всё равно запустится, просто откроется в браузере по умолчанию
вместо отдельного окна.

## Как это устроено

- `desktop/launcher.py` — при запуске готовит `.env` (генерирует уникальный
  `SECRET_KEY`), применяет миграции базы данных, поднимает Django на свободном
  локальном порту и открывает интерфейс либо в нативном окне (через `pywebview`),
  либо в браузере, если нативное окно недоступно на данной системе.
- При первом запуске автоматически создаётся администратор для входа в `/admin/` —
  логин и одноразовый пароль будут показаны в окне консоли. Сохраните пароль,
  он больше нигде не отображается.
- Регистрация обычных бригад — через саму страницу приложения (кнопка
  «Начать бесплатно»), отдельный аккаунт не нужен.
- Закрытие окна приложения (или консоли, если используется браузерный режим)
  останавливает локальный сервер.

## Известные ограничения десктоп-режима

- База данных — SQLite (не PostgreSQL), это осознанное упрощение для локального
  использования одним пользователем на одном компьютере. Для многопользовательской
  работы используйте серверный деплой (см. основной `README.md`).
- Функции, требующие внешние сервисы (ЮKassa, WhatsApp/Telegram, ФНС и т.д. —
  см. ТЗ, Addendum №1/№2), в десктоп-режиме недоступны без соответствующих
  API-ключей в `.env`.
