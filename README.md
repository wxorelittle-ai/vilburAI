# Вильбур AI

SaaS-платформа на Django для строительных бригад, прорабов и заказчиков ремонта.
Полное техническое задание — см. `Brigadir_Pro_TZ.docx` (консолидированный документ:
основной промпт + Addendum №1 «Адаптация под РФ» + Addendum №2 «Модуль J»).

## Десктоп-версия

Проект можно запустить локально на своём компьютере (Windows/macOS/Linux) одним
установочным файлом, без развёртывания на сервере — см. `desktop/README.md`.
Быстрый старт: `install_windows.bat` (Windows) или `./install_mac_linux.sh` (Mac/Linux).

**Если установка через `pip` не идёт из-за VPN/прокси** (ошибка вида
`Missing dependencies for SOCKS support`) — используйте Docker, см. `DOCKER.md`:

```
docker compose up --build
```

Docker собирает и запускает приложение в изолированной среде, обходя проблемы
с прокси на хост-системе.

## Стек

Django 5.0 · Python 3.12 · PostgreSQL 15+ (SQLite для локальной разработки) ·
Tailwind CSS (собранный статический CSS) · xhtml2pdf + python-docx + openpyxl ·
ЮKassa · Gunicorn + Nginx

**Стили** собраны заранее в `static/css/app.css`. Play CDN не используем: он тянул
~400 КБ JS-компилятора на каждую загрузку и оставлял приложение без стилей офлайн,
что противоречило PWA-требованию раздела 7.5 ТЗ. Пересобрать после правки шаблонов:

```bash
npx tailwindcss@3 -c tailwind.config.js -i static/css/tailwind.src.css -o static/css/app.css --minify
```

## Установка (локально)

```bash
python3 -m venv venv
source venv/bin/activate          # Windows: venv\Scripts\activate
pip install -r requirements.txt

cp .env.example .env
# откройте .env и при необходимости поправьте значения —
# по умолчанию всё работает "из коробки" на SQLite и DEBUG=True

python manage.py migrate
python manage.py createsuperuser
python manage.py runserver
```

Приложение будет доступно на `http://127.0.0.1:8000/`, админка — на `/admin/`.

## Переменные окружения

Полный список — в `.env.example`. Ключевые:

| Переменная | Назначение |
|---|---|
| `SECRET_KEY` | Django secret key (сгенерировать случайную строку для прода) |
| `DEBUG` | `True` локально, `False` в проде |
| `DATABASE_URL` | Если не задана — используется SQLite; в проде — `postgres://user:pass@host:5432/db` |
| `ALLOWED_HOSTS` | Домены, с которых разрешён доступ (через запятую) |
| `YOOKASSA_SHOP_ID` / `YOOKASSA_SECRET_KEY` | Ключи ЮKassa (тестовые на старте, боевые после проверки) |

**Важно:** ключи ЮKassa, пароли БД и `SECRET_KEY` никогда не хранятся в коде — только в `.env`,
который не коммитится в Git (см. `.gitignore`). Это прямое требование раздела 12/16 ТЗ.

## Структура проекта

```
config/          — настройки Django, корневые urls
core/            — Brigada, регистрация, вход, личный кабинет, профиль (Этап 1)
documents/       — Модуль A: генератор документов + Word-экспорт (PRO)
calculator/      — Модуль C: калькулятор себестоимости + Excel-экспорт (PRO)
smety/           — Модуль B: генератор смет, публичные ссылки
billing/         — Тарифы, ЮKassa (с демо-режимом), LimitTracker
desktop/         — Десктоп-лончер (см. desktop/README.md)
deploy/          — Конфигурация продакшен-деплоя (см. deploy/DEPLOY.md)
templates/       — HTML-шаблоны (base.html + шаблоны по приложениям)
static/          — статические файлы проекта (включая шрифты для PDF)
```

## Деплой (Yandex Cloud / Timeweb, согласно ТЗ раздел 9)

Полная инструкция, включая автоматизированный скрипт первичной настройки сервера,
systemd-юнит Gunicorn, конфигурацию Nginx и бэкапы — см. **`deploy/DEPLOY.md`**.

Коротко:

```bash
sudo ./deploy/setup_server.sh   # первичная настройка сервера с нуля
sudo certbot --nginx -d ваш-домен.ru   # SSL
./deploy/deploy.sh              # обновление на последующих релизах
```

Хранение персональных данных — только на серверах в РФ (152-ФЗ); Oracle Cloud, AWS,
Google Cloud исключены (раздел 16 ТЗ).
